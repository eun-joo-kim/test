Homebrew 설치하기 (macOS)

macOS 패키지 관리자 - 개발 도구 설치의 필수품

🍺 Homebrew란?

  • macOS용 패키지 관리자
  • 터미널 명령어로 프로그램 설치
  • 개발자 필수 도구
  • Node.js, Git 등 쉽게 설치

🪟 Windows는 Homebrew 불필요 → 다음 슬라이드로

📥 설치 방법

1️⃣ 터미널 열기 (Cmd + Space → "터미널" 검색)

2️⃣ 아래 명령어 복사 & 붙여넣기:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

3️⃣ 비밀번호 입력 → Enter → 완료까지 대기

✅ 설치 확인

brew --version → Homebrew 4.x.x
✓ 이 메시지가 나오면 성공!

Node.js 설치하기

CLI 도구 설치를 위한 필수 사전 준비

🍎 macOS 설치

# Homebrew로 설치 (권장)

brew install node

# 또는 공식 사이트에서 다운로드

nodejs.org → LTS 버전 다운로드

Homebrew 없으면: brew.sh 에서 설치

🪟 Windows 설치

1️⃣ nodejs.org 접속

2️⃣ LTS 버전 다운로드 클릭

3️⃣ 설치 파일 실행 → Next 클릭

4️⃣ 설치 완료!

✅ npm이 자동으로 함께 설치됨

⚡ 설치 확인은 PowerShell에서 실행하세요!

✅ 설치 확인 방법

node --version → v20.x.x
npm --version → 10.x.x

🪟 Windows에서 오류가 발생한다면?

PowerShell에서 스크립트 실행이 차단되어 있을 수 있습니다. 아래 명령어를 먼저 실행하세요:

Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

💡 이유: Windows는 보안상 기본적으로 스크립트 실행을 막아둡니다. 이 명령어는 현재 사용자에 한해 신뢰할 수 있는 스크립트 실행을 허용합니다.

Gemini CLI 설치하기

💡 왜 Gemini CLI인가?

  • 빠름: 응답 속도가 빠름
  • 쉬움: Google 계정만 있으면 OK
  • 에이전트 LLM: 에이전트의 두뇌로 사용
💡 사전 준비: Node.js가 설치되어 있어야 합니다 → node --version 으로 확인

Gemini API 키 발급받기

Gemini CLI는 무료지만, 인증을 위해 API 키가 필요합니다

🔵 Google API 키 발급 방법

  1. aistudio.google.com 접속
  2. Google 계정으로 로그인
  3. Get API Key 클릭
  4. Create API Key 클릭
  5. 프로젝트 선택 (없으면 생성)
  6. ⚠️ 키 복사 (한 번만 표시!)

💡 왜 API 키가 필요할까?

  • 일반 API: 토큰 사용량에 따라 과금
  • Gemini CLI: 터미널에서 사용하면 무료!
  • 무료지만 인증은 필요 → API 키 발급
  • 에이전트의 LLM으로 Gemini 사용

✅ CLI로 쓰면 무료니까 걱정 마세요!

🔐 API 키는 비밀번호와 같습니다. 절대 다른 사람에게 공유하지 마세요!

터미널에서 API 키 설정

환경변수로 API 키를 저장하면 매번 입력할 필요가 없습니다

🍎 macOS 설정

# .zshrc 파일에 영구 저장

echo 'export GEMINI_API_KEY="AIza..."' >> ~/.zshrc

# 설정 적용

source ~/.zshrc

✅ 확인: echo $GEMINI_API_KEY

🪟 Windows 설정

1️⃣ Win + R → sysdm.cpl 입력

2️⃣ 고급 탭 → 환경 변수 클릭

3️⃣ 사용자 변수에서 새로 만들기

4️⃣ 변수 이름: GEMINI_API_KEY

　 변수 값: AIza...

✅ 확인: echo %GEMINI_API_KEY%

💡 Windows PowerShell에서: $env:GEMINI_API_KEY 로 확인 가능!

핵심: 내가 직접 만드는 게 아니라, AI에게 요청하는 것!

1단계: 아이디어 정하기

AI와 대화하며 아이디어를 발굴하세요

  1. 1. AI에게 자기소개하기
    "나는 ~하는 사람이야"
  2. 2. AI에게 질문해달라고 요청
    "뭘 만들면 좋을지 질문해줘"
  3. 3. AI 질문에 답변하기
    답할 수 있는 것만 답하면 OK
  4. 4. 추천받고 수정하며 확정
    "이건 좀 다르게..." → 반복

💬 시작 프롬프트 예시

나는 [직업/역할]이고,

[회사/팀]에서 일해.

업무에 활용할 에이전트를 만들고 싶은데,

뭘 만들면 좋을지 질문해줘.

✅ 아이디어 확정 후 → PRD 요청!

2단계: AI에게 PRD 요청하기

아이디어가 확정되면, PRD(기획서)를 요청합니다

👤 1단계에서 아이디어 확정 후:

"좋아, 이 아이디어로 갈게!"

👤 PRD 요청:

"이 아이디어로 PRD를 md 파일로 만들어줘"

📄 AI가 만들어주는 PRD

  • ✅ 프로젝트 개요
  • ✅ 목표 및 배경
  • ✅ 입력/출력 정의
  • ✅ 기능 요구사항
  • ✅ 평가 기준
  • ✅ 기술 스택 제안

💡 수정이 필요하면 피드백 후 재요청!

3단계: AI에게 프롬프트 설계 요청

에이전트가 사용할 프롬프트도 AI가 만들어줍니다

👤 프롬프트 요청:

"이 PRD 기반으로 에이전트 프롬프트를 설계해줘"

🤖 AI가 설계해주는 것:

  • • 역할 정의 (당신은 ~전문가입니다)
  • • 작업 설명 (~를 생성하세요)
  • • 규칙/제약 (반드시 ~해야 합니다)
  • • 출력 형식 (~형식으로 출력하세요)

💡 피드백으로 개선하기

"역할을 좀 더 구체적으로 해줘"

"규칙에 글자수 제한을 추가해줘"

"출력 예시도 포함해줘"

→ 원하는 프롬프트가 될 때까지 반복!

4단계: AI에게 코드 구현 요청

코드도 AI가 만들어줍니다

👤 코드 요청:

"이 PRD 기반으로 에이전트 코드를 만들어줘"

🤖 AI가 해주는 것:

코드 생성 → 파일 저장 → 실행 → 디버깅

👤 테스트 요청:

"실행해서 테스트해봐"

💡 피드백으로 개선하기

"에러가 났어, 고쳐줘"

"출력 형식을 바꿔줘"

"속도를 더 빠르게 해줘"

→ 원하는 결과가 될 때까지 반복!
