핵심 개념

PRD란?

Product Requirements Document = 제품 요구사항 문서

쉽게 말해, "AI에게 주는 설계도"

뭘 만들지, 어떻게 작동해야 하는지
AI가 이해할 수 있게 정리한 문서

📐 비유: 설계도가 명확할수록 결과물이 정확해진다

전통적 PRD vs 바이브코딩 PRD

전통적 PRD 바이브코딩 PRD
모호해도 개발자가 해석 구체적이어야 AI가 이해
"직관적인 UI" OK ✓ "버튼 색상 #3B82F6, 라운드 8px" 필요
암묵적 요구사항 OK 모든 것을 명시해야 함
개발자가 빈틈 채움 AI는 빈틈을 임의로 채움
바이브코딩에서 PRD = "소스 코드" 그 자체!

AI-Friendly PRD의 7가지 필수 섹션

1. 프로젝트 개요

무엇을 왜 만드는가

2. 목표와 성공 지표

측정 가능한 목표

3. 사용자 스토리

누가, 무엇을, 왜

4. 기능 요구사항

구체적인 기능 목록

5. 기술 스택

사용할 기술 (모르면 AI에게!)

6. 제약사항 (NOT)

하지 말아야 할 것

7. 참고자료

PRD 작성 실전 예시

SNS 마케팅 카피 생성 에이전트

## 프로젝트 개요
제품 정보를 입력하면 SNS 채널별 맞춤 마케팅 카피를 자동 생성하는 에이전트

## 핵심 기능
1. 제품명, 특징, 타겟 입력 받기
2. 인스타그램/페이스북/블로그용 카피 각각 생성
3. 톤앤매너 선택 (친근한/전문적/유머러스)
4. 해시태그 자동 추천

## 입출력 명세
- Input: {제품명, 핵심특징 3개, 타겟층, 톤앤매너}
- Output: {인스타 카피 150자, 페북 카피 300자, 블로그 제목+인트로}

## 성공 지표
- 3개 채널 카피를 10초 이내 생성
- 각 카피에 해시태그 5개 이상 포함

실전: AI에게 PRD 요청하기

이 프롬프트를 복사해서 사용하세요

나는 [직업/역할]이고, [회사/상황]에서 일해.

[반복하는 업무]를 자동화하는 에이전트를 만들고 싶어.

아래 형식으로 PRD를 작성해줘:

1. 한 문장 설명
2. 입력 → 출력
3. 좋은 예시 / 나쁜 예시

💡 AI가 만든 PRD를 검토하고, 부족한 부분은 피드백해서 수정하세요

꿀팁 #1: 기술스택 모를 때

AI에게 물어보세요!

나는 [SNS 마케팅 카피 생성 에이전트]를 만들려고 해.
비개발자인데, 이걸 바이브코딩으로 만들고 싶어.

아래 조건에 맞는 기술스택을 리스팅해줘:
- 초보자도 이해하기 쉬운 것
- 무료 배포 가능한 것
- 각 선택지의 장단점과 특징도 알려줘
💡 AI는 여러 선택지를 리스팅해주고, 장단점과 특징을 비교해줍니다!

알면 PRD 쓰기 쉬워지는 개념들 (1/2)

몰라도 되지만, 알면 AI에게 더 정확히 요청할 수 있어요

예: "Python으로 만들어줘"

예: "내 PDF 기반으로 답변"

💾

데이터베이스 (DB)

예: "고객 정보를 DB에 저장"

🔄

워크플로우

예: "입력 → 처리 → 출력 순서"

🤖

예: "GPT-4 모델 사용해줘"

알면 PRD 쓰기 쉬워지는 개념들 (2/2)

이런 용어를 알면 더 정확하게 요청할 수 있어요

📤

입력/출력 (I/O)

예: "키워드 입력 → 글 출력"

🔁

반복 (Loop)

예: "10개 제품에 대해 반복"

⚡

트리거

예: "이메일 오면 자동 실행"

📝

프롬프트

AI에게 주는 지시문
예: "시스템 프롬프트 설정해줘"

🔗

연동/통합

다른 서비스와 연결
예: "슬랙/노션과 연동해줘"

💡

핵심

개념 이름만 알아도
AI가 알아서 구현해줌!
"RAG 적용해줘" 한마디면 OK

모호하게 vs 구체적으로

같은 요청도 어떻게 쓰느냐에 따라 결과가 달라집니다

❌ 모호한 프롬프트

"마케팅 이메일 써줘"

→ AI: "어떤 제품? 누구한테? 몇 글자?"
→ 질문 10번 주고받기
→ 결국 원하는 게 아님

✅ 구체적인 프롬프트

"20대 여성 타겟, 화장품 신제품 홍보,
친근한 톤, 300자 이내, CTA 버튼 포함"

→ AI가 바로 이해
→ 수정 1-2번으로 완성
→ 원하는 결과물!

구체적일수록 AI가 질문 없이 바로 작업한다!

피해야 할 실수 3가지

1. 너무 모호함

❌ "좋은 글 써줘"

✅ "20대 직장인을 위한 300자 재테크 팁"

2. 예시 없음

❌ "적절하게 써줘"

✅ "이런 느낌으로: [예시 첨부]"

3. 제한 없음

❌ "알아서 해줘"

✅ "가격 언급 금지, 500자 이내"

황금 규칙: AI가 질문 없이 바로 작업할 수 있을 만큼 구체적으로!

꿀팁 #2: 반복적 PRD 개선 전략

AI와 대화하며 PRD 발전시키기

[1차] 초안 요청

"X 에이전트 PRD 초안 작성해줘"

[2차] 개선 요청

"빠진 부분이나 모호한 부분 지적하고 개선해줘"

[3차] 문제점 확인

"이 PRD로 구현할 때 예상되는 문제점은?"

[4차] 최종 확정

"최종 PRD + 구현 단계별 체크리스트 만들어줘"

💡 포인트

• 한 번에 완벽한 PRD는 없다

• 대화하며 점점 좋아진다

• AI가 빈틈을 찾아준다

• 3-4회 반복이면 충분

에이전트 PRD 실전 예시

AI 이메일 분류 에이전트

## 에이전트 역할
받은 이메일을 자동으로 분류하고 우선순위를 매기는 AI 비서

## 핵심 기능
1. 이메일 내용 분석 (발신자, 제목, 본문)
2. 카테고리 분류: 긴급/업무/개인/스팸/뉴스레터
3. 우선순위: 상/중/하
4. 요약 생성 (3줄 이내)

## 입출력 명세
- Input: 이메일 JSON {from, subject, body, date}
- Output: {category, priority, summary, suggested_action}

## 실패 처리
- 분류 확신도 < 70%: "미분류"로 처리
- API 에러: 3회 재시도 후 로깅

🔢 단계별 구현 순서 명시하기

AI는 한 번에 다 만들면 헤맨다

❌ 나쁜 예

"사용자 인증 시스템 만들어줘"

→ AI가 어디서부터 시작할지 모름

✅ 좋은 예

Phase 1: DB 스키마 (users 테이블)
Phase 2: API (/login, /register)
Phase 3: 인증 로직 (JWT 토큰)
Phase 4: UI (로그인 폼)
💡 핵심: 의존성 순서대로 → 기초부터 쌓기

✅ 테스트 가능한 완료 조건

"되면 됐지" 대신 명확한 기준 (Acceptance Criteria)

❌ 모호한 기준

"빠르게 동작해야 함"

"에러 처리 잘 해야 함"

✅ 테스트 가능한 기준

"페이지 로딩 2초 이내"

"잘못된 이메일 → '올바른 이메일을 입력하세요' 표시"

Given-When-Then 형식:

Given: 사용자가 로그인 페이지에 있을 때
When: 잘못된 비밀번호 입력
Then: "비밀번호가 틀렸습니다" 메시지 표시

⚠️ 엣지 케이스 미리 정의하기

AI에게 물어보기: "뭐가 잘못될 수 있을까?"

"이 기능에서 뭐가 잘못될 수 있을까?
엣지 케이스 5개 알려줘"

🌐 인터넷 끊겼을 때

📭 빈 값 입력했을 때

👆 동시에 여러 번 클릭했을 때

📝 아주 긴 텍스트 입력했을 때

🔒 권한 없는 사용자가 접근할 때

💡 미리 정의하면 → AI가 알아서 처리 코드를 만들어준다!

🎭 AI에게 역할 부여하기

더 전문적인 답변을 원한다면

기본 요청

"로그인 기능 만들어줘"

역할 부여 요청

"너는 보안을 중요시하는 시니어 백엔드 개발자야.

SQL 인젝션, XSS 공격을 방지하는
안전한 로그인 기능을 만들어줘."
🛡️ "보안 전문가"
♿ "접근성 전문가"
⚡ "성능 최적화 전문가"
👶 "초보자용 선생님"

✂️ 작은 단위로 쪼개기

AI에게 단계를 쪼개달라고 요청하세요!

❌ 너무 큰 단위

"쇼핑몰 만들어줘"

→ AI가 중간에 길을 잃음

✅ AI에게 쪼개달라고 요청

"쇼핑몰을 만들건데,
단계를 쪼개서 알려줘"

→ AI가 단계별로 정리해줌

🤖 AI의 응답 예시:

1. 상품 목록 페이지

2. 상품 상세 페이지

3. 장바구니 기능

→ 각 단계별로 하나씩 개발 진행!
