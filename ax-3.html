02

왜 코드 편집기인가?

New Workflow
1. 자료 수집
폴더에 PDF, 문서, 이미지 모으기
↓
2. AI와 결합
"이 자료들 분석해줘" 한 마디로
↓
3. 계획 → 생성
구조 설계 후 HTML/PDF/PPT 출력
😫 기존 방식의 문제
  • 파일이 여기저기 흩어짐
  • 자료 찾기 어려움
  • 버전 관리 불가능
✨ 코드 편집기 방식
  • 하나의 작업 공간에서 모든 것
  • 검색 한 번에 전체 탐색
  • Git으로 버전 관리

프롬프트 작성의 핵심

AI에게 명확하게 알려줘야 할 것들
🎯
용도
이걸 왜 만드는가?
내부 공유? 외부 발표? 제안서?
👥
대상
누구를 위한 것인가?
개발자? 기획자? 경영진?
📄
결과물
어떤 형태로 나와야 하는가?
HTML? PDF? PPTX?
💡
"잘 만들어줘"보다 "누가, 왜, 어떤 형태로"가 있으면 AI가 똑똑해집니다
08

결과물 형태 정하기

아웃풋이 명확해야 AI가 잘 만든다
출력 형식
  • 웹 HTML로 발표할 건지?
  • PPTX 파일로 뽑아야 하는지?
  • HTML → PDF로 내보내기?
규격과 방향
  • A4 세로형 vs 가로형
  • 16:9 슬라이드 vs 4:3
  • 모바일 대응 필요 여부
📐
규격을 명확히 하면 수정 횟수가 줄어듭니다

구조 명칭을 알면 소통이 쉬워진다

상단·하단 및 네비게이션
헤더 (Header)
최상단 영역. 슬라이드 번호나 문서 제목
푸터 (Footer)
최하단 영역. 페이지 번호, 날짜, 저작권
프로그레스 바
현재 위치를 보여주는 막대/화살표 지표
GNB
Global Navigation Bar. 전체 메뉴 바
LNB
Local Navigation Bar. 섹션 내 세부 목차
💡 TIP
용어를 알면 의사소통 효율 200% UP!

레이아웃과 여백 용어

📏
마진 (Margin)
가장자리와 콘텐츠 사이
바깥 여백
📦
패딩 (Padding)
박스 안 텍스트와 테두리 사이
안쪽 여백
↔️
거터 (Gutter)
단과 단 사이
간격 여백
🔲
그리드 (Grid)
보이지 않는
격자 가이드라인
✨
"마진 넉넉히" → 시원한 느낌 / "그리드에 맞춰주세요" → 딱딱 정렬
12

콘텐츠 구성 요소 용어

키 비주얼 (Key Visual)
표지나 핵심 슬라이드의 메인 이미지
불렛 포인트 (Bullet)
나열식 문장 앞의 기호 (•, -, >)
콜아웃 (Callout)
특정 부분을 강조하는 설명 상자
캡션 (Caption)
사진이나 도표 아래 짧은 설명글
디바이더 (Divider)
영역을 구분하는 선
위계 (Hierarchy)
크기와 굵기로 중요도 표현
📝
헤드라인 → 서브헤드 → 바디카피 : 제목 → 부제목 → 본문
13

프롬프트 예시

이 슬라이드를 만들 때 실제로 요청한 내용
# 1. 먼저 MD로 내용 정리 요청
"slides-content.md 기반으로 전체 슬라이드를 만들어주세요"

# 2. 디자인 방향 제시
"모던하고 구조적으로 이해가 잘 되게 만들어주세요"

# 3. 구체적인 효과 요청
"첫 장표 배경에 그라데이션이 움직이는 우주 같은 효과"
"프로그레스 바와 GNB도 만들어주세요"
"호버 효과를 만들어주세요"

# 4. 세부 조정
"콘텐츠 박스 간의 여백이 별로입니다" → 수정
"슬라이드 마진 조금만 더 넓게" → 수정

아이디어부터 시작해도 된다

완벽한 계획 없이도 시작할 수 있다
  • 발표 내용을 잘 못 정하겠으면?
  • 아이디어를 늘어놓고 "HTML 먼저 만들어줘" 요청
  • HTML을 보면서 고쳐가며 수정하는 것도 좋은 방법
  • 빈 곳에서 시작하는 것보다 함께 만들어가기
🤝
AI와 핑퐁하듯이 주고받으며 완성해 나가세요
16

HTML로 만들면 좋은 점

🎨
디자인 자유도
  • CSS로 어떤 디자인이든
  • 애니메이션, 전환 효과
  • 인터랙티브 요소 추가
  • 프레임워크 제약 없음
🔗
공유와 접근성
  • 링크 하나로 공유
  • 설치 없이 브라우저만
  • 모바일에서도 OK
  • 업데이트 즉시 반영
📄
다양한 출력
  • 웹에서 바로 발표
  • PDF로 내보내기
  • PPTX 변환 가능
  • 하나의 소스 → 여러 포맷

정리

  1. 코드 편집기 = 자료 관리 + AI 협업의 최적 환경
  2. 프롬프트에는 용도, 대상, 결과물 형태를 명확히
  3. 디자인은 스타일 키워드로 표현
  4. 슬라이드 구조 용어를 알면 소통이 쉬워진다
  5. MD로 내용 먼저 → HTML로 변환
  6. HTML은 디자인, 효과, 공유 모든 면에서 유리
